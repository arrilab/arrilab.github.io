v0.0.0, 2024-06-06 블로그 목표 설정 중

기본

  1. 목표 : 블로그와 포트폴리오 역할을 해야한다.
  2. 가치관 : 블로그와 포트폴리오에 개성, 가치관, 철학이 반영되어있어야 한다.
  3. 디자인 : 디자인은 미적인 부분보다 가독성과 UXUI에 최적화한다.
  4. 최적화 : 최적화 및 SEO도 개선 한다.
  5. 수익 : 블로그에 광고를 달지 않는다. 차라리 후원을 받는다.
  6. 백업 : 자료는 백업이 용이하며 유사시 다른 플랫폼으로 이전 할 수 있어야 한다. (글 내용은 md로 작성)
  7. 기술 : 템플릿을 사용하기 보다는 직접 만들어서 web 개발 경험 용도로도 사용한다

블로그

  1. 주제 : 주제는 3가지로 한정한다. 회사에서의 개발자, 기술자로서의 개발자, 동료로서의 개발자
  2. 내용 : 단순 자료 수집함이 아닌 문제와 아이디어 해결의 스토리, 객관적인 근거와 수치, 개인적인 연구와 경험 가장 중요한 것은 전달하고자 하는 의도가 있어야 한다.
  3. 내용 : 한번 보고 말 자료가 아닌 계속 생각나서 다시 보고 싶은 글을 작성한다.
  4. 내용 : 개발 기술을 가르치는 것은 블로그보다는 유튜브나 책 등을 이용한다.
  5. 갯수 : 글 갯수가 많으면 오히려 글 하나의 가치가 떨어진다. 무의미한 글은 제거하고 수정해 핵심만 남긴다. 독자의 시간 또한 소중하다 (50개 이내 유지)

포트폴리오

  1. 포트폴리오는 내가 한 것, 하고 있는것, 할 것이 담겨있어야 하며, 내 역할과 기술들이 명확하게 있어야 한다.
  2. 포트폴리오를 보는 대상은 면접관을 전제로 하며 면접관은 비개발직군(CEO등)과 개발직군(개발자,CTO,아키텍쳐)를 따로 가정해서 고령한다.
